Executive Summary

한 장으로 정리한 사업 개요.

문제

주차 시 차량 앞에 노출되는 전화번호·집주소가 보이스피싱·스토킹 등 범죄의 개인정보 소스가 된다.

솔루션

NFC 태그에 UUID만 저장하고, 서버가 실제 번호를 가상번호로 마스킹해 중계한다. 앱 설치 없이 브라우저만으로 연결된다.

브랜드

"팍씨 (Mr. PARK)" — 급한 순간의 추임새와 주차(park), 한국인 성씨(박)의 삼중 의미.

핵심 지표

현 주차 연락 방식의 구조적 문제

한국인은 대시보드에 전화번호를 올려둔 채 주차한다. 이 관행은 세 가지 구조적 문제를 가진다.

개인정보 노출

번호 = 범죄 소스

보이스피싱 · 스토킹 · 스팸 발신의 1차 소스. 번호는 한 번 노출되면 회수 불가.

주소 · 신상 유출

동·호수까지 노출

아파트 주차 시 "OO동 OO호 차주" 메모가 관행. 차량 번호와 결합 시 역추적 용이.

연결 실패

모르는 번호 회피

보이스피싱 경계로 다수가 모르는 번호 수신 거부. 실제 연결률은 낮다.

이 문제는 수십 년간 방치되어 왔다. "번호 써붙이면 된다"는 관행의 반복이 곧 시장이 열려있다는 증거다.

"NFC = 결제 해킹" 우려에 대한 답변

패시브 NFC 태그로는 결제 탈취가 구조적으로 불가능하다. 실제 위협은 소셜 엔지니어링 영역에 존재한다.

구조적 차단

패시브 태그 = 결제 불가

  • 결제 NFC(삼성·애플페이)는 EMV 프로토콜 기반
  • POS → 폰 명령 → 보안칩(SE) 응답 → 생체 인증 필수
  • 태그는 수동적 읽기 대상, 폰을 제어할 수 없다

악성 하드웨어 시나리오

PN532 칩 + 배터리 + 블루투스 결합 시 카드 형태 기기 제작은 이론상 가능하나, Secure Element는 OS 수준에서 격리되어 생체 인증 없이 결제 불가하다.

실제 위협 · 소셜 엔지니어링

위협내용
피싱 페이지 유도결제처럼 보이는 가짜 페이지로 카드번호 입력 유도
딥링크 악용kakaopay://send?amount=5000 형태로 결제앱 실행 + 금액 선입력
악성 앱 설치 유도APK 다운로드 페이지 연결
최종 결제 승인은 항상 사용자 생체 인증을 거친다. 따라서 진짜 위협은 사용자를 직접 속이는 소셜 엔지니어링이다. 본 서비스는 페이지 구조를 전화 버튼 단일 요소로 한정해 해당 공격 표면을 제거한다.

경쟁 우위 · 해자 구성 요소

특허는 보조 장치이며, 실질 해자는 네트워크 · 데이터 · 브랜드 · 스위칭 코스트의 4축이다.

특허 방향 (보조 장치)

  • NFC 태그 기반 익명 차량 연락 시스템
  • 주차 태그 위변조 감지 방법
  • 관리소 차량 인증 시 개인정보 최소화 방법

* 기술 특허보다 프로세스 특허 (방법 특허) 중심으로 출원.

표준화 전략 (톨게이트 이론)

  1. 먼저 깔리는 것 — 보급 속도 우선
  2. 개방형으로 보이는 것 — "앱 없어도 된다"
  3. API 개방 — 경쟁자를 파트너로 전환

실질 해자 4축

① 네트워크 효과

스캔 경험을 가진 사람이 차주로 전환되어 태그를 구매한다.

② 데이터 자산

주차 분쟁 패턴 데이터가 보험·관리 업체에 가치.

③ 브랜드 선점

"주차 NFC = 팍씨"의 카테고리 연상 선점.

④ 스위칭 코스트

아파트 단지 단위 계약 후 태그 교체는 실물 교체 비용 발생.

① 차량 1대 · 운전자 N명

  • 연락 우선순위: 동시 · 순차 · 시간대별 발신
  • 차주 앱 "지금 내가 운전 중" 토글로 실시간 전환
  • 법인차: 관리자가 드라이버 풀 등록 → 당일 배정자 자동 연결

② 태그 없는 차량 연결 (번호판 역방향)

방법 A · 위치 알림 푸시

차주가 자기 태그를 스캔해 주변 차량에 "내 차 위치" 푸시 발송

방법 B · 번호판 기반 조회 (핵심)

발신자가 상대 번호판을 입력 → 서버가 UUID 조회 → 익명 연결 성립

닭-달걀 문제의 반전. 태그 보급률이 0%여도 서비스가 먼저 사용되며, 그 수요가 태그 도입 동기를 만든다.

리스크 매트릭스 및 대응

기술 리스크는 낮으나 시장·경쟁 리스크가 높다. 각 축에 대한 구체적 대응 방안을 수립하였다.

항목리스크
기술 구현 낮음
수익화 · PMF 높음
경쟁 진입 높음
규제 · 개인정보 중간
대체재 (주차 로봇 등) 낮음

* 대체재 (주차 로봇)는 커버리지가 신축 · 대형 건물에 한정되어, 골목·구축·빌라 영역에서는 영향 미미.

대응 방안

  • 수익화 · 번호판 역방향 플로우로 보급률 이전에도 매출 성립
  • 경쟁 · 속도 · 브랜드 선점 · API 개방으로 후발 주자를 파트너화
  • 규제 · 안심번호 API에 얹어 법적 회색 지대 회피
  • 기술 · 오픈 레일 사용, 데모 45분 · 출시 단기화
  • 대체재 · 로봇 주차장에도 차량 ID 인증용 태그 적용 가능
리스크 전반은 속도로 완화할 수 있다. 시장 선점 시점이 곧 방어선이 된다.
