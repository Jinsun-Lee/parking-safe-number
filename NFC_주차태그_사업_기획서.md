## 1. 개요 및 기술 과제

**아이디어 핵심:**
- 주차 차량 앞 전화번호/집주소 노출 → 범죄 위험
- NFC 태그로 대체, 번호 랜덤 생성 → 차주 번호 비노출로 연결
- 앱으로 NFC 태그 업데이트 가능
- 피싱(결제 유도) 방어 필요
- 아파트 관리용: 등록 차량 여부만 확인 (개인정보 비노출)

## 2. 기술 과제 리스트

### 핵심 구성 요소

**익명 전화 연결 (번호 마스킹)**
- 랜덤 가상번호 발급 및 실제 번호와 매핑하는 서버 구축
- 통신사 API 또는 중계 서비스 연동 (네이버 클라우드 Simple&Safe, 카카오 안심번호 API)
- 통화 종료 후 세션 만료 처리

**NFC 태그 발급 및 관리**
- NFC 태그에 고유 UUID 기록 (전화번호 직접 저장 금지)
- UUID → 가상번호 매핑을 서버에서만 관리
- 앱에서 NFC 태그 재프로그래밍 기능 (NDEF 쓰기)
- 태그 분실/훼손 시 UUID 무효화 및 재발급 플로우

**피싱 방어**
- 태그 스캔 시 앱 내부에서만 동작하도록 딥링크 + 앱 강제 실행 구조
- Android App Links / iOS Universal Links로 앱 바인딩
- 태그에 결제 트리거 삽입 불가하도록 NFC 페이로드 타입 화이트리스트 검증
- 태그 위변조 감지: 서버 측 서명값 검증 (HMAC)

**아파트 관리용 인증 태깅기**
- 관리소 전용 단말기에서 태그 스캔 시 "등록 차량 여부"만 반환
- 차량 등록 DB와 연동, 응답은 Yes/No + 동/호수 마스킹
- 관리소 단말기 인증 토큰 별도 발급 (일반 앱과 권한 분리)

## 3. 앱 없이 전화 연결 — 핵심 플로우

**브라우저 기반 (앱 없어도 작동)**

```
NFC 스캔
  → 브라우저 자동 실행
  → https://call.도메인/uuid=a3f9c2
  → 서버: UUID → 가상번호 조회
  → 웹페이지에 [전화 걸기] 버튼 표시
  → 버튼 클릭 → tel:010-XXXX-XXXX 발신
  → 차주 실제번호로 중계
```

스마트폰이면 안드로이드/iOS 상관없이, 앱 없이도 작동.

**피싱 방어 (앱 강제 실행 포기 → 서버 검증으로 대체)**
- 가짜 태그: 우리 도메인 아니면 아무것도 없는 페이지
- URL 위조: HMAC 서명 검증으로 불가
- 결제 유도: 우리 페이지는 전화버튼 하나뿐

## 4. NFC 태그 — 무엇을 살 것인가

### 태그 선택
| 태그 | 가격 | 추천 이유 |
|---|---|---|
| NTAG213 | 개당 200~400원 | 가장 범용, 안드로이드/iOS 전부 호환, 데모에 충분 |
| NTAG215 | 개당 400~700원 | 용량 큼, 지금 단계엔 불필요 |
| NTAG424 DNA | 개당 2,000~4,000원 | 동적 서명 가능, 데모엔 오버스펙 |

**→ 데모는 NTAG213으로 충분**

구매처:
- 알리익스프레스: "NTAG213 NFC sticker" 검색, 50개에 5~10달러
- 네이버쇼핑: "NFC 스티커 태그 NTAG213" 검색

### 데모 제작 순서
1. NFC Tools 앱 설치 → URL 쓰기 → 태그에 갖다 대기 (10분)
2. Vercel + 단순 HTML 한 페이지, 전화버튼 하나 (30분)
3. 다른 폰으로 스캔 테스트 (5분)

## 5. NFC 결제 피싱 분석

### 패시브 NFC 태그로 결제 불가 — 이유
- 결제 NFC(삼성페이/애플페이)는 완전히 다른 프로토콜
- POS 단말기 → 폰에 EMV 명령 → 보안칩(Secure Element) 응답 → 생체인증 필수
- 태그는 폰이 읽는 것, 태그가 폰을 제어하는 게 아님

### 실제 위협
| 위협 | 설명 |
|---|---|
| 피싱 페이지 유도 | 결제처럼 생긴 가짜 페이지로 카드번호 입력 유도 |
| 딥링크 악용 | kakaopay://send?to=XXX&amount=5000 같은 URL로 결제앱 실행 + 금액 선입력 |
| 악성 앱 설치 유도 | APK 다운로드 페이지로 연결 |

**핵심: 최종 승인은 항상 사용자 생체인증 필요. 사용자가 속아서 직접 인증하는 소셜 엔지니어링이 진짜 위협.**

### 악성 하드웨어 가능성
- PN532 칩(5달러) + 배터리 + 블루투스 → 신용카드 두께로 제작 가능
- Rogue POS 형태로 EMV 명령 전송 시도 가능
- 그러나 보안칩(Secure Element)은 OS 레벨 격리 → 생체인증 없이 결제 불가

**우리 시스템 대응: 오브젝트에 배터리 공간 없는 구조, 앱에서 능동/수동 NFC 감지**

## 6. 투자자 시각 비판적 분석

### 핵심 의문
1. **수익화**: "그냥 번호 써붙이면 되지" 저항 → "번호 털린다"가 핵심 세일즈 포인트
2. **B2B vs 개인**: 개인 소비자 먼저, B2B는 순차 공략
3. **진입장벽**: 특허보다 브랜드 선점 + 네트워크 효과가 핵심
4. **안심번호 API 의존**: 초기엔 네이버 클라우드 임차, 장기적 내재화 검토
5. **왜 아직 없나**: iOS NFC 개방이 2019년, 안심번호 B2C 개방 최근, 시장 선점 타이밍

### 리스크 등급
| 항목 | 리스크 |
|---|---|
| 기술 구현 | 낮음 |
| 수익화 | 높음 |
| 경쟁 | 높음 |
| 규제 | 중간 |


## 10. 톨게이트 분석 — 병목 장악

**핵심 인사이트:**
- 톨게이트의 본질 = "통과하지 않으면 안 되는 지점" 장악

**표준이 되기 위한 조건:**

## 13. 차주가 전화를 걸어야 할 때 (역방향 플로우)

**방법 1:** 내 태그 스캔 → 앱 실행 → "내 차 위치 알림 보내기" → GPS 기반 주변 차량에 푸시

**방법 2 (더 중요):** 앱에서 상대 번호판 입력 → 서버에서 UUID 조회 → 익명 연결
- 태그 없는 차량도 번호판으로 연결 가능
- 보급률 0%여도 서비스 가능, 태그 도입 동기 생성

## 14. 장기 로드맵

```
지금:     태그 팔고 통화 중계
↓
중기:     차량 연락 표준 플랫폼
↓
장기:     차량 ID 플랫폼
           → 연락 / 주차비 정산 / 사고 당사자 연결
           → 보험 연동 / 차량 인증
↓
최종:     물류 시장
           → 택배 차량 이중주차 민원
           → 화물차 하역 지연 연락
           → 배달 오토바이 소속 확인
           → 쿠팡, CJ대한통운이 우리 API 사용
```

**엔드게임:**
> 차량 한 대 = 디지털 ID 하나
> 번호판청 다음으로 차량 정보를 가진 민간 플랫폼
> 국내 등록차량 2,500만대 × 3,000원 = 750억원 시장

## 15. 주차 로봇 위협 분석

**위협 맞음, 그러나:**
- 로봇 주차 커버리지: 신축 스마트 주차장, 대형 건물 전용
- 골목 이중주차, 구축 아파트, 빌라 밀집 지역 = 10~20년 후에도 그대로
- 로봇이 커버하는 건 전체 주차 문제의 5% 미만

**더 중요한 관점:**
> "의미 없어도 붙이고 다닌다"
> 에어팟 케이스 스티커처럼 기능을 넘어 아이덴티티가 되면 대체 불가
> 오히려 로봇 주차장에도 차량 ID 인증용으로 태그 부착 가능
