05 · Security

NFC 결제 피싱 분석

패시브 NFC 태그로 결제 불가

  • 결제 NFC(삼성페이·애플페이)는 완전히 다른 프로토콜
  • POS → 폰에 EMV 명령 → 보안칩 응답 → 생체인증 필수
  • 태그는 폰이 읽는 것, 태그가 폰을 제어하는 게 아님

악성 하드웨어 가능성

  • PN532 칩(5달러) + 배터리 + 블루투스 → 신용카드 두께 제작 가능
  • Rogue POS 형태로 EMV 명령 전송 시도 가능
  • 그러나 보안칩(Secure Element)은 OS 레벨 격리 → 생체인증 없이 결제 불가

실제 위협

위협설명
피싱 페이지 유도결제처럼 생긴 가짜 페이지로 카드번호 입력 유도
딥링크 악용kakaopay://send?to=XXX&amount=5000 URL로 결제앱 실행 + 금액 선입력
악성 앱 설치APK 다운로드 페이지로 연결
최종 승인은 항상 사용자 생체인증 필요.
사용자가 속아서 직접 인증하는 소셜 엔지니어링이 진짜 위협
대응: 오브젝트에 배터리 공간 없는 구조, 앱에서 능동·수동 NFC 감지
