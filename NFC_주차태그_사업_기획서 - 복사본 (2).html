10
Security · Investor Q&A

"NFC = 결제 해킹 아닙니까?" · 아니다

패시브 태그로 결제 불가 — 구조적으로

  • 결제 NFC(삼성·애플페이)는 완전히 다른 프로토콜
  • POS → 폰 EMV 명령 → 보안칩 응답 → 생체인증 필수
  • 태그는 폰이 읽는 것, 태그가 폰을 제어하는 게 아님

악성 하드웨어 가능성도 막힘

  • PN532 + 배터리 + 블루투스 = 카드 두께 제작 가능
  • Rogue POS 형태로 EMV 명령 시도 가능
  • 보안칩은 OS 격리 → 생체인증 없이 결제 불가

실제 위협은 소셜 엔지니어링

위협설명
피싱 페이지결제처럼 생긴 가짜 페이지
딥링크kakaopay://send?amount=5000
악성 앱APK 다운로드 유도
최종 승인은 항상 생체인증.
사용자가 속아서 직접 인증하는 게 진짜 위협.
우리 페이지 = 전화 버튼 하나.
속을 여지가 없다.
